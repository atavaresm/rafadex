### Task 1: Fix the evolution strip page-overflow bug

**Files:**
- Modify: `style.css` (`html, body` rule; `.evo-strip` rule)

**Interfaces:**
- Produces: no page-level horizontal scroll regardless of how many evolution branches a Pokémon has (Eevee's 8-branch chain is the worst case in the dataset).

- [ ] **Step 1: Add the fix**

In `style.css`, find:

```css
html, body { margin: 0; background: var(--bg); color: var(--ink);
  font-family: "Baloo 2", system-ui, sans-serif; overscroll-behavior: none; }
```

Replace with:

```css
html, body { margin: 0; background: var(--bg); color: var(--ink);
  font-family: "Baloo 2", system-ui, sans-serif; overscroll-behavior: none;
  overflow-x: hidden; }
```

Then find:

```css
.evo-strip { display: flex; align-items: center; gap: 6px; margin: 10px 0;
  overflow-x: auto; max-width: 100%; }
```

Replace with:

```css
.evo-strip { display: flex; align-items: center; gap: 6px; margin: 10px 0;
  overflow-x: auto; max-width: 100%; min-width: 0; }
```

(Root cause: `.evo-strip` is a flex item inside `.detail`, itself a flex container. Flex items default to `min-width: auto`, refusing to shrink below their content's intrinsic width — Eevee's 8 thumbnails + 7 arrows are ~810px wide, blowing out `.detail`/`#app`'s ~500px available width with nothing upstream clipping it. `min-width: 0` lets the strip shrink to fit, so its own pre-existing `overflow-x: auto` takes over correctly. The `overflow-x: hidden` on `html, body` is a defensive backstop so no future layout bug can cause page-level horizontal scroll again.)

- [ ] **Step 2: Live verification (the exact regression this fixes)**

Serve the app (`python3 -m http.server 8000`), open `#dex/133` (Eevee, 8-branch chain). In the browser console:

```javascript
`scrollWidth: ${document.documentElement.scrollWidth} | innerWidth: ${window.innerWidth} | overflow: ${document.documentElement.scrollWidth > window.innerWidth}`
```

Expected: `overflow: false` (scrollWidth ≤ innerWidth — no page-level horizontal scroll). Confirm the strip itself still scrolls horizontally within its own bounds (drag left inside `.evo-strip` — later branches become visible). Repeat the `scrollWidth`/`innerWidth` check on `#dex/4` (Charmander, 3-branch chain) as a no-regression check — should also be `false`.

### Task 2: Scroll position restore + sticky detail header

**Files:**
- Modify: `app.js` (`renderRoute` at the bottom of the file)
- Modify: `style.css` (`.topbar` rule)

**Interfaces:**
- Produces: `scrollPositions` (module-level object, hash string → scrollY), `previousHash` (module-level, last-rendered hash string or `null`). Returning to a previously-visited hash restores its last scroll position instead of resetting to the top. `.topbar` is `position: sticky` everywhere it's used (type world, detail, game share the same `topbar()` helper) — always visible/tappable during scroll.

- [ ] **Step 1: Add scroll restore to `renderRoute`**

In `app.js`, find:

```javascript
function renderRoute() {
  document.body.style.background = "";
  Sound.stopSpeech();
  const [route, arg] = location.hash.replace(/^#/, "").split("/");
  window.scrollTo(0, 0);
  if (route === "type") renderType(arg);
  else if (route === "dex") renderDetail(Number(arg));
  else if (route === "game") renderGame();
  else renderHome();
}
window.addEventListener("hashchange", renderRoute);
renderRoute();
```

Replace with:

```javascript
const scrollPositions = {};
let previousHash = null;

function renderRoute() {
  if (previousHash !== null) scrollPositions[previousHash] = window.scrollY;
  document.body.style.background = "";
  Sound.stopSpeech();
  const [route, arg] = location.hash.replace(/^#/, "").split("/");
  if (route === "type") renderType(arg);
  else if (route === "dex") renderDetail(Number(arg));
  else if (route === "game") renderGame();
  else renderHome();
  window.scrollTo(0, scrollPositions[location.hash] || 0);
  previousHash = location.hash;
}
window.addEventListener("hashchange", renderRoute);
renderRoute();
```

(The scroll-to-top call moves from *before* the new screen renders to *after* — restoring a non-zero saved position only works once the new content is tall enough to actually be scrolled into. A hash with no saved entry falls back to `0`, identical to today's behavior.)

Then add this new function anywhere at the top level of `app.js` (e.g. right after `renderParentPanel`):

```javascript
function normalizeSearch(text) {
  return text.normalize("NFKD").replace(/[̀-ͯ]/g, "").toLowerCase();
}

function renderSearchPanel() {
  const panel = el("div", "search-panel");
  const input = el("input", "search-input");
  input.type = "text";
  input.placeholder = "Buscar por nome...";
  panel.append(input);
  const results = el("div", "search-results");
  panel.append(results);

  input.oninput = () => {
    const query = normalizeSearch(input.value.trim());
    results.innerHTML = "";
    if (!query) return;
    const matches = window.DEX.filter(m => normalizeSearch(m.name).includes(query)).slice(0, 20);
    for (const mon of matches) {
      const row = el("button", "search-result bounce",
        `<img src="${sprite(mon.id, "thumb")}" alt=""><span>${mon.name}</span>`);
      row.onclick = () => { contextIds = window.DEX.map(m => m.id); go(`#dex/${mon.id}`); };
      results.append(row);
    }
  };
  return panel;
}
```
